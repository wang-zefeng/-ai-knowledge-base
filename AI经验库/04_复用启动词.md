# 复用启动词

## 用途

每次打开 ChatGPT、龙虾、Codex、WorkBuddy、Claude 时，可以复制这里的启动词，让 AI 快速理解我的项目经验和 Skill。

## 推荐先读

如果 AI 可以访问文件，请先让它读取：

1. `00_总入口.md`
2. `01_项目总览.md`
3. `项目经验/<项目名>/项目状态.md`
4. 必要时再读取对应的 `经验卡片.md` 或 `Skill卡片.md`

---

## 通用项目启动词

```md
你现在要基于我的 AI 经验库继续推进项目。

请按顺序读取和理解：

1. 00_总入口.md
2. 01_项目总览.md
3. 和本次任务相关的 项目经验/<项目名>/项目状态.md
4. 必要时再读取对应的 经验卡片.md 或 Skill卡片.md

工作要求：

- 先总结你理解的当前项目状态
- 再判断本次任务属于项目推进、经验沉淀还是 Skill 复用
- 不要重构项目
- 不要大改 UI
- 不要新增复杂权限系统
- 不要改无关文件
- 每次只推进一个明确任务
- 如果产生可复用经验，最后告诉我应该沉淀到哪个文件

本次任务：

【填写任务】
```

---

## 对话沉淀提示词

```md
请判断下面内容是否值得沉淀进我的 AI 经验库。

如果不值得，回复：

不用沉淀。

如果值得，请判断它属于哪一种：

1. 项目状态
2. 项目经验卡片
3. 项目 Skill 卡片
4. 通用 Skill 卡片
5. 待整理

然后按对应格式整理。

要求：

- 不要复述聊天过程
- 只保留以后能直接复用的内容
- 内容要极简、清楚、可执行
- 提示词要能直接复制使用
- 坑点要明确
- 未验证内容必须标注"待验证"
- 最后告诉我应该保存到哪个文件

内容如下：

【粘贴待整理内容】
```

具体需求：

1.
2.
3.

## WorkBuddy 任务拆解启动词

```md
你现在是这个项目的产品经理和任务调度员。

请根据下面需求拆成 GitHub Issue。

要求：

- 每个 Issue 只解决一个明确问题
- 每个 Issue 要包含目标、范围、实现要求、验收标准
- 不要让 Codex 重构项目
- 不要让 Codex 大改 UI
- 优先保证当前版本可内测
- 每个 Issue 最后都要附上给 Codex 的执行提示词

项目背景：

【填写项目背景】

本次需求：

【填写需求】

请输出：

1. Issue 标题
2. Issue 目标
3. 实现范围
4. 具体要求
5. 验收标准
6. 优先级
7. 给 Codex 的执行提示词
```
