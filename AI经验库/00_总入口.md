# AI 经验库总入口

## 用途

这是给 ChatGPT、龙虾、Codex、WorkBuddy、Claude 使用的项目经验入口。

每次开启一个新对话时，优先从这里理解：

1. 当前有哪些项目
2. 每个项目推进到哪里
3. 应该读取哪些经验卡片或 Skill 卡片
4. 本次任务应该如何沉淀回经验库

## 推荐读取顺序

### 1. 先读项目总览

读取：

`01_项目总览.md`

目的：

- 判断当前任务属于哪个项目
- 快速了解每个项目的当前状态
- 确认下一步应该推进什么

### 2. 再读具体项目状态

读取：

`项目经验/<项目名>/项目状态.md`

目的：

- 理解当前目标
- 理解最近完成了什么
- 理解当前阻塞和下一步任务
- 避免重复发散、重复设计、重复推翻

### 3. 按需读取项目经验或 Skill

如果任务和某个项目强相关，读取：

- `项目经验/<项目名>/经验卡片.md`
- `项目经验/<项目名>/Skill卡片.md`

如果任务是跨项目通用方法，读取：

- `通用Skill/Skill卡片.md`

### 4. 临时内容先放待整理

如果对话中出现有价值但还没归类的内容，先放入：

`待整理/临时记录.md`

后续再判断它应该沉淀为：

- 项目经验卡片
- 项目专用 Skill
- 通用 Skill
- 不需要沉淀

## 使用原则

1. 不要先发散，先读状态。
2. 不要重构已有项目，先确认当前目标。
3. 不要把所有内容都沉淀，只有能复用的才沉淀。
4. 项目进度写进 `项目状态.md`。
5. 可复用经验写进 `经验卡片.md`。
6. 可重复执行的方法写进 `Skill卡片.md`。
7. 临时对话和未判断内容写进 `待整理/临时记录.md`。

## 给 AI 的启动提示词

```md
你现在要基于我的 AI 经验库继续推进项目。

请按顺序读取和理解：

1. 00_总入口.md
2. 01_项目总览.md
3. 和本次任务相关的 项目经验/<项目名>/项目状态.md
4. 必要时再读取对应的 经验卡片.md 或 Skill卡片.md

工作要求：

- 先总结你理解的当前项目状态
- 再判断本次任务属于项目推进、经验沉淀还是 Skill 复用
- 不要重构项目
- 不要大改 UI
- 每次只推进一个明确任务
- 如果产生可复用经验，最后告诉我应该沉淀到哪个文件

本次任务：

【填写任务】
```
