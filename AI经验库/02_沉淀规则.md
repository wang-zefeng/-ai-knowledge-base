# 经验沉淀规则

## 用途

这份规则用来判断一段对话、一次开发过程、一个提示词或一个解决方案是否值得放进 AI 经验库。

目标不是保存所有聊天记录，而是保存以后可以直接复用的判断、流程和提示词。

## 什么值得沉淀

### 1. 项目经验卡片

适合保存某个项目里已经验证过的结论。

例如：

- 某个功能应该怎么做
- 某个方案为什么被放弃
- 某个坑下次要避免
- 某个模块当前最小可行路径是什么
- 某个项目的关键业务判断是什么

保存位置：

`项目经验/<项目名>/经验卡片.md`

### 2. 项目 Skill 卡片

适合保存某个项目专用、可以反复执行的方法。

例如：

- EC-Copilot 的竞品分析流程
- OmniFlow 的任务拆解流程
- 三科的成分卖点提炼流程

保存位置：

`项目经验/<项目名>/Skill卡片.md`

### 3. 通用 Skill 卡片

适合保存跨项目可复用的方法。

例如：

- 如何把对话沉淀成经验卡片
- 如何写 Codex 开发提示词
- 如何拆 GitHub Issue
- 如何排查部署失败

保存位置：

`通用Skill/Skill卡片.md`

### 4. 项目状态

适合保存项目当前进度，而不是经验。

例如：

- 当前目标
- 最近完成
- 当前阻塞
- 下一步任务
- 重要决策

保存位置：

`项目经验/<项目名>/项目状态.md`

## 什么不值得沉淀

以下内容默认不沉淀：

- 一次性的闲聊
- 没有结论的发散想法
- 不能复用的临时操作
- 已经过期的错误判断
- 没有验证过却被当成结论的内容
- 大段原始对话

如果暂时不确定，先放到：

`待整理/临时记录.md`

## 沉淀时的压缩原则

1. 不复述聊天过程。
2. 只保留结论、步骤、提示词、坑点和复用方式。
3. 每张卡片只解决一个明确问题。
4. 能复制使用的提示词要完整。
5. 未验证内容必须标注“待验证”。
6. 已废弃内容不要删除结论来源，但要标注“已废弃”。

## 快速判断提示词

```md
请判断下面内容是否值得沉淀进我的 AI 经验库。

如果不值得，回复：

不用沉淀。

如果值得，请判断它属于哪一种：

1. 项目状态
2. 项目经验卡片
3. 项目 Skill 卡片
4. 通用 Skill 卡片
5. 待整理

然后按对应格式整理。

要求：

- 不要复述聊天过程
- 只保留以后能直接复用的内容
- 内容要极简、清楚、可执行
- 提示词要能直接复制使用
- 坑点要明确
- 未验证内容必须标注“待验证”
- 最后告诉我应该保存到哪个文件

内容如下：

【粘贴待整理内容】
```
