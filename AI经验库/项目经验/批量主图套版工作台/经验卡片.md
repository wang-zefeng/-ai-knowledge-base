# 批量主图套版工作台 - 经验卡片

## 项目经验

### 主图套版 MVP 先拆数据层，不先追求最终设计自动化

结论：第一版应验证「活动线框 + 产品资料 + 机制数据 -> 主图 PNG」的闭环，而不是让 AI 或复杂设计系统直接生成最终主图。

可复用做法：
- 把主图拆成背景层、产品层、机制层、文案层、线框层。
- 用结构化 `brief` 管理产品、活动、文案、赠品、右侧内容槽和模板 ID。
- 用模板 JSON 管理槽位、尺寸、顺序和字段绑定。
- CSV / Excel 只负责输入产品和活动机制，渲染逻辑负责把字段放到固定槽位。
- 产品图和赠品图先用本地文件名精确匹配，避免第一版引入脆弱爬虫。
- 导出前必须预检缺字段、缺图片、模板 ID 和文案长度。

已验证事实：
- 当前源码是无依赖静态 MVP，可直接打开 `index.html`。
- `src/app.js` 已包含 CSV 解析、批量行映射、产品素材文件名匹配、预检、Canvas 渲染和 PNG 导出。
- `data/product-lineframe-template.json` 已把双11预售固定分层模板拆成可解释字段和槽位。

坑点：
- 不要把完整源码、`.git`、素材、导出图片或日志直接塞进经验库；经验库只存状态、链接、索引和可复用结论。
- 本地工作区没有 git remote，源码仓库链接要用用户确认的 <https://github.com/wang-zefeng/batch-main-image>。
- AI 后续只适合生成可审核的结构化卖点/机制字段，不适合绕过人工检查直接出最终主图。
