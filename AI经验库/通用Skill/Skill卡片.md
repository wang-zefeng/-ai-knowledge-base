## Skill：搭建 AI 经验库

### 用途
在本地创建一个跨项目共享的 Markdown 经验沉淀库，按项目维度组织，让所有 AI 会话都能自动发现和使用。

### 适用场景
- 需要长期积累多个项目的 AI 协作经验
- 想让不同 AI 工具（WorkBuddy、ChatGPT、Claude、Codex）都能读写同一个经验库
- 不想用数据库、不想做网页，只要纯 Markdown

### 输入
- 一个固定目录路径（推荐 `C:\Users\<用户>\WorkBuddy\ai-knowledge-base\`）
- 已有或计划中的项目列表

### 执行步骤
1. 创建永久固定目录（不要放在临时会话目录里）
2. 搭建目录结构：`待整理/` + `项目经验/<项目名>/`（含经验卡片、Skill卡片、项目状态）+ `通用Skill/`
3. 写入入口文档：00_总入口、01_项目总览、02_沉淀规则、04_复用启动词
4. 写入 `~/.workbuddy/MEMORY.md`，声明库的固定路径和沉淀规则
5. Git init + 推送 GitHub，保证跨设备可用
6. 新增项目时，在 `项目经验/` 下新建文件夹并写入 3 个空模板即可

### 输出结果
一个固定路径的 Markdown 经验库，任何 WorkBuddy 会话自动识别，换设备 `git clone` 即可复用。

### 可复用提示词
```
帮我搭建一个 AI 经验库，要求：
1. 放在固定位置 C:\Users\...\ai-knowledge-base\
2. 按项目维度分文件夹，每个项目含经验卡片、Skill卡片、项目状态
3. 通用 Skill 放单独文件夹
4. 写入口文档和沉淀规则
5. 写入用户级 MEMORY.md 让所有会话自动识别
6. 不做网页、不接数据库、不用向量库，纯 Markdown
```

### 注意事项
- 必须放在固定路径，不能放临时会话目录，否则其他会话找不到
- 用户级 MEMORY.md（`~/.workbuddy/MEMORY.md`）是让所有 AI 会话自动发现的关键
- 其他 AI 工具（非 WorkBuddy）不会自动发现，需要用户手动粘贴启动词
- 推 GitHub 时需要 token 有 repo 权限，否则只能手动建仓库再 push

### 示例
见 `C:\Users\Administrator\WorkBuddy\ai-knowledge-base\AI经验库\` 当前结构。

### 成熟度
可复用

## Skill：沉淀一下自动分类并同步 Git

### 用途
把一次项目进展、对话结论、开发流程或可复用方法，压缩成经验库卡片，并自动同步到 GitHub。

### 适用场景
- 用户说“沉淀一下”“沉淀经验”“存到经验库”“更新项目状态”等类似指令
- 一次任务完成后，需要把可复用结论保存下来
- 发现了坑点、流程、提示词、项目状态变化或可复用工作方法

### 输入
- 当前对话中已经验证过的结论、步骤、提示词、坑点或项目状态
- 如果涉及具体项目，需要知道项目名

### 执行步骤
1. 在 `C:\Users\Administrator\WorkBuddy\ai-knowledge-base` 执行 `git pull`。
2. 读取 `AI经验库\02_沉淀规则.md`，判断写入类型。
3. 按规则分类到项目经验、项目 Skill、项目状态、通用 Skill 或待整理。
4. 只写可复用结论、步骤、提示词、坑点和状态，不粘贴大段原始聊天。
5. 写入后执行 `git add -A`、`git commit -m "沉淀: <简短描述>"`、`git push`。

### 输出结果
经验库中新增或更新一条结构化 Markdown 记录，并且远端 GitHub 仓库同步成功。

### 可复用提示词
```md
沉淀一下。

请先读取 AI经验库\02_沉淀规则.md，判断当前内容应该进入项目经验、项目 Skill、项目状态、通用 Skill 还是待整理。
只保留以后能复用的结论、步骤、提示词、坑点或状态，不要复述聊天过程。
写入后自动 git add、commit、push。
```

### 注意事项
- 不要把 API Key、Token、Cookie、账号密码、隐私数据或未审查的长聊天记录推到 GitHub。
- 未验证的结论必须标注“待验证”。
- 如果无法判断项目归属，先写入 `待整理/临时记录.md`。
- 如果 Git push 失败，要明确说明失败原因，不要假装同步成功。

### 示例
用户说“沉淀一下刚才这个部署排查流程”，AI 应读取沉淀规则，将可复用步骤写入 `通用Skill/Skill卡片.md` 或对应项目经验卡片，然后提交并推送。

### 成熟度
可复用
