# 通用 Skill 卡片库

## 用途

保存跨项目可反复复用的能力、方法、流程和工作流。不限于某个具体项目。

## 适合保存的 Skill

1. 需求拆解 Skill
2. Codex 开发提示词 Skill
3. WorkBuddy 调度 Skill
4. AI 项目复盘 Skill
5. GitHub Issue 拆分 Skill
6. 表格解析需求设计 Skill
7. 高保真 Demo 反推产品逻辑 Skill
8. 部署排查 Skill
9. 经验沉淀 Skill
10. 业务模块梳理 Skill

## Skill 卡片格式

```md
## Skill：标题

### 用途
这个 Skill 是用来解决什么问题？

### 适用场景
什么时候用它？

### 输入
使用这个 Skill 前，需要准备什么资料？

### 执行步骤
1.
2.
3.
4.

### 输出结果
执行完应该得到什么？

### 可复用提示词
可以直接复制给 AI 的提示词。

### 注意事项
使用时容易出什么问题？

### 示例
给一个简单例子。

### 成熟度
试用中 / 可复用 / 已稳定
```

## Skill：沉淀一下自动分类并同步 Git

### 用途
把一次项目进展、对话结论、开发流程或可复用方法，压缩成经验库卡片，并自动同步到 GitHub。

### 适用场景
- 用户说“沉淀一下”“沉淀经验”“存到经验库”“更新项目状态”等类似指令
- 一次任务完成后，需要把可复用结论保存下来
- 发现了坑点、流程、提示词、项目状态变化或可复用工作方法

### 输入
- 当前对话中已经验证过的结论、步骤、提示词、坑点或项目状态
- 如果涉及具体项目，需要知道项目名

### 执行步骤
1. 在 `C:\Users\Administrator\WorkBuddy\ai-knowledge-base` 执行 `git pull`。
2. 读取 `AI经验库\02_沉淀规则.md`，判断写入类型。
3. 按规则分类到项目经验、项目 Skill、项目状态、通用 Skill 或待整理。
4. 只写可复用结论、步骤、提示词、坑点和状态，不粘贴大段原始聊天。
5. 写入后执行 `git add -A`、`git commit -m "沉淀: <简短描述>"`、`git push`。

### 输出结果
经验库中新增或更新一条结构化 Markdown 记录，并且远端 GitHub 仓库同步成功。

### 可复用提示词
```md
沉淀一下。

请先读取 AI经验库\02_沉淀规则.md，判断当前内容应该进入项目经验、项目 Skill、项目状态、通用 Skill 还是待整理。
只保留以后能复用的结论、步骤、提示词、坑点或状态，不要复述聊天过程。
写入后自动 git add、commit、push。
```

### 注意事项
- 不要把 API Key、Token、Cookie、账号密码、隐私数据或未审查的长聊天记录推到 GitHub。
- 未验证的结论必须标注“待验证”。
- 如果无法判断项目归属，先写入 `待整理/临时记录.md`。
- 如果 Git push 失败，要明确说明失败原因，不要假装同步成功。

### 示例
用户说“沉淀一下刚才这个部署排查流程”，AI 应读取沉淀规则，将可复用步骤写入 `通用Skill/Skill卡片.md` 或对应项目经验卡片，然后提交并推送。

### 成熟度
可复用
